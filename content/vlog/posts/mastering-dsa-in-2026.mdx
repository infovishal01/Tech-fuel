## Problem-Solving Framework

### The 5-Step Approach

```
1. Understand the Problem
   ├─ Read carefully
   ├─ Identify input/output
   └─ Clarify edge cases

2. Think of Brute Force
   ├─ Simplest solution
   ├─ Time/Space complexity
   └─ Why it's inefficient

3. Optimize
   ├─ Look for patterns
   ├─ Use appropriate data structures
   └─ Apply algorithms

4. Implement
   ├─ Write clean code
   ├─ Handle edge cases
   └─ Optimize for readability

5. Test
   ├─ Test with examples
   ├─ Test edge cases
   └─ Analyze complexity
```

## Coding Best Practices

### 1. Input Validation
```python
# Always validate inputs
def two_sum(nums, target):
    if not nums or len(nums) < 2:
        return []
    # ... rest of the code
```

### 2. Edge Case Handling
```python
# Handle edge cases explicitly
if not head:
    return None
if not head.next:
    return head
```

### 3. Time Complexity Analysis
```python
# O(n) time, O(n) space
def contains_duplicate(nums):
    seen = set()
    for num in nums:
        if num in seen:
            return True
        seen.add(num)
    return False
```

### 4. Space Optimization
```python
# O(1) space solution for two sum (if sorted)
def two_sum_sorted(nums, target):
    left, right = 0, len(nums) - 1
    while left < right:
        current_sum = nums[left] + nums[right]
        if current_sum == target:
            return [left + 1, right + 1]
        elif current_sum < target:
            left += 1
        else:
            right -= 1
    return []
```

## Common Mistakes to Avoid

### 1. Jumping to Coding
- **Problem**: Starting to code without understanding the problem
- **Solution**: Spend 5-10 minutes understanding the problem first

### 2. Ignoring Edge Cases
- **Problem**: Code works for examples but fails on edge cases
- **Solution**: Always test with empty input, single element, duplicates

### 3. Not Analyzing Complexity
- **Problem**: Not knowing the time/space complexity of your solution
- **Solution**: Always analyze and mention complexity

### 4. Poor Variable Naming
- **Problem**: Using names like `x`, `y`, `temp`
- **Solution**: Use descriptive names like `current_sum`, `max_length`

### 5. Not Testing Thoroughly
- **Problem**: Assuming code is correct without testing
- **Solution**: Write test cases, use Tech Fuel's test runner
